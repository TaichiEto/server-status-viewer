# StatusNode (ホスト直接実行版)

共用サーバーのリソース状況（CPU/GPU/RAM/SSH/Process）を、ホストから直接リアルタイムで監視・可視化するための軽量ツールです。Dockerを使用しないため、サーバー上の全ユーザーの活動や全てのプロセスを完全に把握できます。

## 🚀 主な機能

- **リソース監視**: ホスト全体のCPU、RAM使用率をグラフ表示。
- **GPU詳細**: NVIDIA GPUの型番、使用率、VRAM（使用量/合計）、温度を網羅。
- **SSH接続監視**: 現在ログイン中のユーザー名、端末、**接続元IPアドレス**を一覧表示。
- **プロセス管理**: 全ユーザーのプロセスを一覧表示し、特定のユーザーでフィルタリング可能。
- **クイックアクション**: 
  - ブラウザから「Killコマンド」や「通知コマンド」をワンクリックでコピー。
  - ターミナルに貼り付けるだけで、即座にプロセスの停止やユーザーへの連絡が可能。

## 🛠 動作環境

- **OS**: Linux (Ubuntu, Debian, CentOS 等)
- **Runtime**:
  - Go 1.20 以上
  - Node.js 18 以上 (ビルド用)
- **System Utilities**: `who`, `procps`, `nvidia-smi` (GPU監視用)

## 📦 インストールと起動

### 1. フロントエンドのビルド
```bash
cd frontend
npm install
npm run build
cd ..
```
※ `dist/` ディレクトリが生成され、Goサーバーがこれを静的ファイルとして配信します。

### 2. サーバーの起動!!!
```bash
go mod tidy
go run main.go
```

### 3. アクセス
ブラウザを開き、以下のURLにアクセスしてください。

- **URL**: `http://<サーバーのIP>:2000`

## ⚠️ 注意事項

- 本ツールはポート `2000` を使用します。ファイアウォール（ufw, iptables等）で許可されているか確認してください。
- 他人のプロセスをKillしたり通知を送ったりするには、実行するユーザー（あなた）が適切な権限（sudo権限など）を持っている必要があります。
- インターネットに公開する場合は、Nginx等のリバースプロキシでパスワード認証をかけることを強く推奨します。
